SQL aggregate functions summarize data across multiple rows by producing a single result per group, or a single result for the whole dataset if no grouping is specified. Common aggregates include `COUNT`, `SUM`, `MIN`, `MAX`, and `AVG`, along with statistical functions like `STDDEV`, `MEDIAN`, and `VARIANCE`. Aggregates normally operate on all input values but can also process only unique values using `DISTINCT`, while `ALL` explicitly includes every row. The `GROUP BY` clause allows aggregates to be calculated for subsets of rows sharing the same value, producing one result per group, and all non-aggregated columns in the `SELECT` clause must appear in the `GROUP BY`. Row filtering before grouping is done with `WHERE`, while filtering results produced by aggregate calculations requires the `HAVING` clause because it operates after grouping. SQL can also generate hierarchical or multidimensional subtotals using extensions like `ROLLUP`, which produces progressive totals and a grand total, and `CUBE`, which computes aggregates for every combination of grouped columns.

Aggregate functions compute a value across multiple rows and normally reduce the result to one row per group using `GROUP BY`, while **analytic (window) functions** perform similar calculations but preserve all original rows by using the `OVER()` clause. Analytic functions can divide data into logical groups with `PARTITION BY` (similar to grouping but without collapsing rows) and can calculate ordered metrics such as running totals, averages, or rankings using `ORDER BY`. When ordering is used, a **windowing clause** defines which rows participate in the calculation (for example running totals, sliding windows, or ranges relative to the current row). Because analytic functions are processed after the `WHERE` clause, filtering by their results requires a subquery or CTE. In addition to aggregate-based analytics (like `SUM`, `AVG`, or `COUNT` with `OVER`), SQL also provides dedicated analytic functions such as `ROW_NUMBER`, `RANK`, `DENSE_RANK` for ranking rows, `LAG` and `LEAD` for accessing previous or next rows, and `FIRST_VALUE` and `LAST_VALUE` for retrieving boundary values within a window.
